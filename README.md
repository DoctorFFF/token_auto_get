# 🚀 AutoCheckin - 极简插件化自动签到系统
轻量级 · 可扩展 · 零依赖 · 纯本地部署

一个基于 Python + Flask 的多站点自动签到工具。无需数据库，采用纯 JSON 文件存储，支持 Nuitka 打包为单文件 EXE，双击即可运行。专为个人用户设计，兼顾隐私安全与极致易用性。

## 📖 使用指南
### 1. 添加账号
在对应站点卡片下方，输入用户名和密码。 点击「添加账号」，数据将自动保存至 data/[插件名].json。

提示：首次使用前，请确保已在目标网站完成注册。本工具不负责自动注册。

### 2. 执行签到
单账号签到：点击账号行右侧的 签到 按钮。
批量签到：勾选多个账号复选框，点击顶部 选中账号签到。
全量签到：点击站点卡片右上角的 该站点全部签到。
模式选择：批量签到时会弹出模态框，可选择：

🚀 并发模式：多线程同时执行，速度极快，适合无严格风控站点。
🐢 顺序模式：逐个执行，更稳定。
固定间隔：手动输入秒数（如 50），每个账号间固定等待该时间。
随机间隔：留空不填，系统会自动在每个账号间生成 2 到 5 秒 的随机等待时间，模拟真实用户操作。

### 3. 查看数据与历史
点击账号行的 详情 按钮，展开查看详细统计。

历史记录：查看每日签到获得的配额值。
图表分析：自动生成「签到额度折线图」和「占比饼图」，追踪长期收益。

### 4. 插件管理
将新的 .py 插件文件放入 plugins/ 目录。
点击页面顶部的「重新扫描插件」按钮，新站点将即时加载，无需重启程序。

## 🔧 插件开发指南
开发一个新站点的签到插件仅需 3 分钟。

在 plugins/ 目录下新建文件，例如 mysite.py。
定义元数据并实现 run 函数：
```python
# plugins/mysite.py

PLUGIN_NAME = "mysite"          # 唯一标识
PLUGIN_TITLE = "My Site"        # 显示名称
PLUGIN_URL = "https://example.com"
PLUGIN_DESC = "这是一个示例插件"

def run(username, password):
    """
    核心签到逻辑
    :return: 标准结果字典
    """
    try:
        # 1. 登录逻辑
        # 2. 签到请求
        # 3. 解析结果
        
        return {
            "success": True,
            "message": "签到成功",
            "data": {
                "checkin_date": "2026-05-25",
                "quota_awarded": 1000,      # 今日获得额度
                "checked_in_today": True,
                "total_checkins": 10,       # 累计签到次数
                "total_quota": 10000,       # 累计总额度
                "records": []               # 历史记录列表（可选）
            }
        }
    except Exception as e:
        return {
            "success": False,
            "message": f"错误: {str(e)}",
            "data": {}
        }
```
保存文件，在前端点击「重新扫描插件」即可生效。

💡 本地调试建议： 项目内置了 testapp_server.py 测试服务器。在开发新插件时，建议先连接本地测试服务器（端口 5001）进行调试。它可以详细打印请求 IP、Header 和耗时，帮助你研究目标网站的风控特征。

### 🚧 正在进行中 (In Progress)
#### 🎯 重点攻坚：图像识别验证码逆向工程
目前部分高价值站点引入了复杂的图像验证码或行为验证（如 Cloudflare Turnstile, Geetest 等），这是自动化签到的最大阻碍。

难点：传统 OCR 难以处理扭曲、噪点干扰严重的动态验证码；部分验证机制涉及前端 JS 加密参数生成。

阶段性成果：
- 预处理算法优化：实现了基于 OpenCV 的二值化、去噪和轮廓提取算法，显著提升了简单字符验证码的识别率。
- 滑块验证模拟：初步完成了滑块轨迹的模拟算法，能够模拟人类鼠标移动特征，绕过简单的轨迹检测。
- OCR 模型接入测试：正在调试集成轻量级深度学习模型（如 PaddleOCR/CNN），针对特定站点的定制字体进行训练，目前测试集识别准确率已达 85%+。

下一步计划：
- 完善对抗性样本处理，提高复杂背景下的识别鲁棒性。
- 研究 Headless Browser (Playwright/Selenium) 与插件系统的无缝结合，以解决纯 HTTP 请求无法处理的 JS 挑战。

### 📅 未来规划

- 增加签到失败自动重试机制
- 支持 Telegram/DingTalk 消息推送通知
